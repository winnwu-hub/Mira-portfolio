# Case 3：商务出差英语 · Storyline 风格交互式课件 — 教学设计文档

> **版本**：v2.0（替换旧版 AI 聊天原型）  
> **定位**：成人职场英语 · 故事驱动 · 三级难度自适应  
> **形态**：HTML 单文件交互式课件（Storyline 风格）  
> **对标岗位**：AI/工具型在线教育公司 Learning Designer

---

## 1. 产品概述

### 产品名称
**BizTrip English** — 商务出差英语

### 一句话描述
一场从北京到新加坡的商务出差，学习者在三个真实情境任务中用英语完成任务，难度自适应。

### 核心价值主张
- **故事驱动**：完整叙事弧线（机场 → 会议 → 晚宴），学习在情境中自然发生
- **三级自适应难度**：同一对话场景提供引导/半自由/自由三种模式，实时切换
- **语域意识培养**：独有「语域适当性」评分维度，训练学习者根据场合切换表达方式
- **脚手架渐进撤除**：即使全程使用引导模式，内部提示也在逐步减少

### 与旧版 Case 3 的区别

| 维度 | 旧版 prototype.html | 新版 storyline-courseware.html |
|------|---------------------|-------------------------------|
| 形态 | 聊天 App 模拟 | Storyline 风格全屏课件 |
| 教学结构 | 隐式（藏在对话中） | 显式（四阶段推进） |
| 难度设计 | 无分级 | 三级实时切换 + 渐进撤除 |
| 评估 | 简版 3 维度 | 语域分析 + 文化贴士 + 递进反馈 |
| 练习 | 无 | 每任务 Prep 阶段小练习 |
| 数据架构 | 硬编码 | JSON 数据驱动 |

## 2. 学习者分析

### 目标用户
- **年龄**：22-40 岁在职成人
- **英语水平**：A0-B1（零基础到中级），课件内部差异化适配
- **学习动机**：工具型动机（职业发展需要）+ 融合型动机（国际商务文化兴趣）
- **学习场景**：碎片化自学，PC 浏览器为主

### 认知特点（成人学习理论依据）
- **Knowles Andragogy**：成人学习者需要「立即能用」的内容；每个场景结束后有 Takeaway 总结
- **自我导向**：学习者可自选难度、自定节奏、可重玩
- **经验关联**：场景设定贴合真实的职场经验（航班延误、客户谈判、商务晚宴）
- **焦虑管理**：零基础学习者从 🟢 引导模式开始，选项式回复消除「不知道说什么」的焦虑

### 水平分层设计

| 层级 | CEFR | 特征 | 课件中的对应模式 |
|------|------|------|-----------------|
| 入门 | A0-A1 | 几乎不会说，需要完整句子提示 | 🟢 引导模式 |
| 初级 | A1-A2 | 能说简单句，词汇和句型有限 | 🟡 半自由模式 |
| 进阶 | A2+ | 能独立组织表达，但语域切换不准 | 🔴 自由模式 |

## 3. 学习目标（Bloom's Taxonomy）

### 总体目标
完成课件后，学习者能够在商务出差的三个典型场景中用英语完成交际任务，并能根据场合选择适当的语域表达。

### 分层目标矩阵

| Bloom 层级 | 具体目标 | 对应环节 |
|-----------|----------|----------|
| **记忆** Remembering | 识别并回忆 20+ 个商务旅行核心词汇 | 📚 Prep 词汇配对练习 |
| **理解** Understanding | 理解航班延误广播、会议议程、餐厅推荐等场景对话 | 🎯 Mission 中听懂 AI 对话 |
| **应用** Applying | 在模拟场景中作出符合交际目标的回复 | 🎯 Mission 对话交互 |
| **分析** Analyzing | 识别自己用语的语域是否与场合匹配 | 📊 Debrief 语域分析 |
| **评价** Evaluating | 对比三个场景的语域差异，反思自己的表达选择 | 📊 Debrief 跨场景对比 |

## 4. 课件架构

### 4.1 整体导航

```
🏠 开始画面（标题 + 情境导入 + 开始按钮）
        │
        ↓
🗺️ 任务中心（3 个任务卡片，线性解锁）
        │
        ├── ✈️ 任务一：机场出行 ──🔒──→ 🤝 任务二：客户会议 ──🔒──→ 🍷 任务三：晚宴社交
        │       │                          │                          │
        │   完成后可重玩              完成后可重玩              完成后可重玩
```

- 线性推进以维护故事弧线
- 已完成任务可随时重玩（点击任务卡片）
- 进度通过 localStorage 持久化

### 4.2 单任务四阶段结构

```
📋 Briefing  →  📚 Prep  →  🎯 Mission  →  📊 Debrief
   (1屏)         (1-2屏)      (核心交互)       (1屏)
```

#### Briefing（任务背景）
- 以第二人称叙述当前情境（「你是 Alex Chen...」）
- 告知任务目标和你会学到什么
- 1 屏，点击「开始准备」进入 Prep

#### Prep（词汇句型准备）
- 核心词汇表（英 + 中 + 🔊 点读发音）
- 关键句型展示（标出句型模板中的可替换槽位）
- 小练习（英→中 词汇配对 / 句型与场景归类）
- 1-2 屏，点击「开始任务」进入 Mission

#### Mission（对话执行）
- 与虚拟 NPC 进行多轮对话（4-6 轮）
- 难度模式实时切换
- 每轮提交后显示 Recast 反馈气泡（🟡 模式）
- 见第 6 节详细设计

#### Debrief（评估与反馈）
- 三维雷达图或进度条评分
- 语域分析 + 文化贴士
- 错误模式总结 + 改进建议
- 「重玩本任务」或「下一个任务」按钮

### 4.3 页面布局

```
┌─────────────────────────────────────────────┐
│  ← 返回任务中心    任务名    当前阶段/总阶段   │  ← 顶栏
├─────────────────────────────────────────────┤
│                                             │
│              场景氛围背景色 + 图标             │
│                                             │
│            ┌───────────────────┐             │
│            │                   │             │
│            │   核心内容区       │             │
│            │   (可变内容)       │             │
│            │                   │             │
│            └───────────────────┘             │
│                                             │
├─────────────────────────────────────────────┤
│   🟢 引导  🟡 半自由  🔴 自由    [操作按钮]  │  ← 底栏
└─────────────────────────────────────────────┘
```

- 难度切换器在 Mission 阶段始终可见；Prep/Debrief 阶段隐藏
- 操作按钮随阶段变化：Briefing →「开始准备」/ Prep →「开始任务」/ Debrief →「重玩」+「下一任务」

## 5. 三个场景 — 内容设计

### 5.1 ✈️ 任务一：机场出行 — Airport Departure

| 维度 | 内容 |
|------|------|
| **情境设定** | 北京首都机场，下午 2 点飞新加坡 SQ807 航班 |
| **交际目标** | 处理航班延误 → 向地勤了解情况 → 通知客户 |
| **NPC** | 👩‍✈️ Ground Staff（地勤人员） |
| **语域** | 事务型（简短直接，信息交换优先） |
| **对话轮次** | 4 轮 |

**Prep 内容**：
- 词汇（6 个）：boarding pass, gate, departure, terminal, check-in counter, delayed
- 句型（3 个）：Which gate for flight...? / Is the flight on time? / My flight has been delayed...
- 小练习：英→中 词汇配对

**Mission 对话流**：
1. 地勤告知航班延误 → 学习者确认延误情况
2. 地勤说明预计延误 1 小时 → 学习者询问新起飞时间
3. 地勤提供具体信息 → 学习者表示理解并询问登机口
4. 地勤通知登机口变更 → 学习者确认并回复

**Debrief 重点**：「旅行生存英语」关键句型总结

---

### 5.2 🤝 任务二：客户会议 — Client Meeting

| 维度 | 内容 |
|------|------|
| **情境设定** | 新加坡陈氏集团办公室，首次见市场 VP 陈女士 |
| **交际目标** | 寒暄开场 → 展示提案 → 回应询价与异议 → 确认下一步 |
| **NPC** | 👩‍💼 Ms. Chen, VP of Marketing（陈女士，市场 VP） |
| **语域** | 正式商务（结构化表达，关系+内容并重） |
| **对话轮次** | 5 轮 |

**Prep 内容**：
- 词汇（7 个）：agenda, proposal, timeline, deliverable, budget, scope, stakeholder
- 句型（4 个）：Thank you for making time... / I'd like to walk you through... / Could you clarify...? / Would a two-week timeline work?
- 小练习：句型 → 会议场景归类（开场/展示/澄清/收尾）

**Mission 对话流**：
1. 陈女士欢迎 → 学习者寒暄开场
2. 陈女士请开始 → 学习者展开提案要点
3. 陈女士询问预算 → 学习者回应报价
4. 陈女士提出异议（timeline 太紧）→ 学习者化解
5. 陈女士表示兴趣 → 学习者确认 next steps

**Debrief 重点**：「商务正式语域」表现分析

---

### 5.3 🍷 任务三：晚宴社交 — Dinner Social

| 维度 | 内容 |
|------|------|
| **情境设定** | 会议成功！陈女士邀请团队在当地餐厅共进晚餐 |
| **交际目标** | 入座寒暄 → 聊菜单 → 回答个人话题 → 敬酒 → 得体告别 |
| **NPC** | 👩‍💼 Ms. Chen + 团队成员 |
| **语域** | 非正式社交（轻松自然，关系建立优先） |
| **对话轮次** | 6 轮 |

**Prep 内容**：
- 词汇（6 个）：cuisine, local specialty, toast, recommend, hosting, cheers
- 句型（4 个）：What's the must-try dish here? / How long have you been with...? / Cheers to... / This has been wonderful.
- 文化贴士：新加坡用餐礼仪（分享式用餐、敬酒顺序、筷子礼仪）
- 小练习：识别正式 vs 非正式表达（语域切换练习）

**Mission 对话流**：
1. 入座 → 学习者寒暄座位
2. 讨论菜单 → 问推荐菜
3. 陈女士问「你怎么入这行的？」→ 学习者回答个人话题
4. 一个关于新加坡 vs 中国饮食习惯的轻松玩笑
5. 敬酒致意 → 学习者举杯说祝酒词
6. 晚餐结束 → 得体告别 + 表达感谢

**Debrief 重点**：「社交语域切换」分析 + 跨文化沟通 tips

## 6. 三级难度与脚手架设计

### 6.1 三级难度对比

| | 🟢 引导模式 (A0-A1) | 🟡 半自由模式 (A1-A2) | 🔴 自由模式 (A2+) |
|---|---|---|---|
| **输入方式** | 点击 3-4 个完整句子选项 | 输入框 + 2-3 个关键词悬浮提示 | 纯空白输入框 |
| **提示量** | 完整选项 | 关键词/短语 | 无 |
| **容错** | 选错 → 即时显示正确句 + 解释 | 提交后 Recast 纠错气泡 | 提交后 Recast + 语域建议 |
| **发音** | 所有选项可 🔊 点读 | 关键词可 🔊 点读 | — |
| **求助** | — | 「展开一个完整选项」 | 「给我关键词提示 💡」 |

### 6.2 实时切换机制

- 底部始终显示三个模式按钮，当前模式高亮
- 切换时：
  - 🟢→🟡：清空输入区，显示关键词提示
  - 🟡→🔴：清空输入区，显示空白输入框
  - 降级方向：不清空已有输入，追加提示
- 切换不打断对话流，不重载当前轮

### 6.3 脚手架渐进撤除（贯穿三个任务）

```
          任务一 ✈️      任务二 🤝      任务三 🍷

🟢 引导   选项带粗体      选项纯文本      选项增至 5-6 个
          关键词高亮      无额外提示      含干扰项

🟡 半自由  3-4 关键词      2 个关键词      仅 1 个模糊提示

🔴 自由   空白输入        空白输入        空白输入
          每轮 1 个 hint   仅求助才有      hint 需点击
```

即使学习者全程留在 🟢 引导模式，内部提示也在逐步减少——实现「不被察觉的难度递进」。

### 6.4 学习理论依据

| 理论 | 在脚手架中的体现 |
|------|-----------------|
| **Vygotsky ZPD** | 三级难度对应三个最近发展区，学习者自选进入哪个区 |
| **Krashen i+1** | 每个模式提供比当前水平略高的输入（选项词汇略高于学习者产出水平） |
| **Swain Comprehensible Output** | 🔴 自由模式强制学习者「被推动去产出」，Debrief 反馈促进注意差距 |
| **Scaffolding 渐进撤除** | 跨任务 + 跨模式双层撤除机制 |

## 7. Mission 对话交互设计

### 7.1 对话界面布局

```
┌─────────────────────────────────────────────────┐
│  ← 返回          任务二：客户会议      3/4      │
├─────────────────────────────────────────────────┤
│                                                 │
│               [场景氛围色背景]                    │
│                                                 │
│  ┌─────────────────────────────────────────┐   │
│  │ 👤 陈女士                                │   │
│  │ ┌─────────────────────────────────┐     │   │
│  │ │ Thank you for coming today.     │ 🔊  │   │
│  │ │ Let's hear your proposal.       │     │   │
│  │ └─────────────────────────────────┘     │   │
│  │                                         │   │
│  │                         💬 You ────┐   │   │
│  │                         │ Thanks for│   │   │
│  │                         │ having me.│🔊🖊️│   │
│  │                         └───────────┘   │   │
│  │                                         │   │
│  │ 👤 陈女士                                │   │
│  │ ┌─────────────────────────────────┐     │   │
│  │ │ Great. What's the timeline      │ 🔊  │   │
│  │ │ you're proposing?               │     │   │
│  │ └─────────────────────────────────┘     │   │
│  │                                         │   │
│  │ ────────── 当前回复轮 ──────────        │   │
│  │                                         │   │
│  │  （输入区：选项 / 关键词+输入框 / 纯输入） │   │
│  └─────────────────────────────────────────┘   │
│                                                 │
├─────────────────────────────────────────────────┤
│   🟢 引导    🟡 半自由    🔴 自由    [提交 →]  │
└─────────────────────────────────────────────────┘
```

### 7.2 关键交互细节

| 元素 | 行为 |
|------|------|
| **AI 气泡出现** | 逐字打字动画（~30ms/字），模拟真人回复节奏 |
| **🔊 语音按钮** | 点击朗读 AI 消息（Web Speech API，`en-US`） |
| **已完成的学习者消息** | 点击 🖊️ 可修改重发（单轮内），培养元认知 |
| **Recast 反馈** | 🟡 模式提交后，正确回复以淡绿色气泡出现在学习者消息下方 |
| **选项悬停** | 微微放大 + 背景色加深 |
| **场景过渡** | 最后一轮结束 → 2 秒静态过渡卡 → Debrief 面板滑入 |

### 7.3 反馈策略（按难度模式）

| 错误情况 | 🟢 引导 | 🟡 半自由 | 🔴 自由 |
|---------|---------|----------|---------|
| 选错选项 | 即时提示 + 正确句高亮 | — | — |
| 输入语法小错 | — | Recast 气泡 + 修正高亮 | Debrief 才提示 |
| 输入词汇错误 | — | Recast 气泡 + 提供目标词 | Debrief 才提示 |
| 输入语域不当 | — | 温和提示「Try a more formal way...」 | Debrief 语域分析 |
| 交际成功 | 肯定 + 推进 | 肯定 + 推进 | 肯定 + 推进 |

反馈递进逻辑：**脚手架越少 → 反馈越延迟 → 越接近真实交际中的自我监控**。

## 8. 评估与反馈设计

### 8.1 三维评分体系

| 维度 | 权重 | 衡量内容 | 数据来源 |
|------|------|----------|----------|
| 🎯 任务完成度 | 40% | 是否完成每轮交际目标 | 关键回复正确率（correct: true 的比例） |
| 📝 语言准确度 | 30% | 词汇和句型使用是否正确 | 🟡/🔴 模式下语法错误计数 |
| 🗣️ 语域适当性 | 30% | 用语是否符合场景语域期望 | 🟡/🔴 模式下正式度匹配检查 |

**语域维度是本课件的评分创新点**：成人职场英语最大的痛点不是「不会说」，而是「不会根据场合说」。这是面试官能一眼看到的差异化教学设计。

### 8.2 三个场景的语域期望

```
机场 ✈️              会议 🤝              晚宴 🍷
事务型                正式商务              非正式社交

✅ "Which gate?"     ✅ "I'd like to       ✅ "Got any
                        walk you through      recommendations?"
✅ "My flight           our proposal."
    has been delayed."                       ✅ "Cheers to a
                     ✅ "Would a two-week       great partnership!"
✅ 简短、直接、信息      timeline work?"
                                             ✅ 轻松、自然、关系
  交换优先           ✅ 结构化、礼貌、           建立优先
                       关系+内容并重          ❌ 过于正式会让
                                            气氛尴尬
```

### 8.3 Debrief 面板内容

1. **总分展示**：三维度进度条或数值
2. **一句话评价**：「你完成了所有关键交际任务。语域选择准确——听起来像个专业的商务旅行者。」
3. **改进建议**：具体到句子的修正（如 `'I want to...' → 试试 'I'd like to...'`）
4. **文化贴士**：场景相关的跨文化沟通知识
5. **下一步**：「重玩本任务」或「下一个任务 →」

### 9.3 数据架构

所有场景内容定义为 JSON 数据，UI 层纯渲染：

```javascript
var SCENES = [
  {
    id: 'airport',
    title: 'Airport Departure',
    icon: '✈️',
    theme: '#2563eb',
    briefing: { /* 情境文本 */ },
    prep: {
      vocabulary: [{ en: '...', zh: '...' }],
      structures: [{ template: '...', slots: ['...'] }],
      exercise: { type: 'match', items: [...] }
    },
    mission: {
      npc: { name: 'Ground Staff', avatar: '👩‍✈️' },
      rounds: [{
        npcLine: '...',
        goal: '...',
        guidedOptions: [           // 🟢
          { text: '...', correct: true/false, feedback: '...' }
        ],
        semiGuidedHints: ['...'],  // 🟡
        autoSpeech: true
      }]
    },
    debrief: { /* 评分 + 反馈 + 文化贴士 */ }
  }
  // ... scene 2, scene 3
];
```

### 9.4 设计原则

- **单文件自包含**：CSS + JS 全部内联，与案例 1、2 保持一致
- **数据驱动**：修改/新增场景只需编辑 SCENES 数组，不碰渲染逻辑
- **事件委托**：统一 `click` 监听 + `data-action` 属性，与已有案例技术模式一致
- **无外部依赖**：纯 vanilla JS，离线可用（除 Web Speech API 需浏览器支持）

## 10. 教学理论支撑汇总

| 理论 | 在课件中的体现 |
|------|--------------|
| **TBLT（任务型教学法）** | 三个任务即三个交际任务，每个任务有明确产出目标 |
| **Krashen i+1** | 所有 AI 输入略高于学习者当前难度层级 |
| **Swain Comprehensible Output** | 🔴 自由模式 + 延迟反馈，推动学习者产出 + 注意差距 |
| **Vygotsky ZPD** | 三级难度对应三个最近发展区 |
| **Knowles Andragogy** | 立即有用的职场内容 + 自我导向 + 经验关联 |
| **Lyster & Ranta 纠错反馈** | Recast + Elicitation（半自由模式的关键词提示） |
| **Bloom's Taxonomy** | 记忆→理解→应用→分析→评价，贯穿四阶段 |
| **SDT（自我决定理论）** | 自主选难度（自主性）+ 评分反馈（胜任感）+ 故事中建立关系（归属感） |
