# AI 听说练习产品设计方案

## SpeakNow — AI 英语对话教练

---

## 一、产品概述

| 项目 | 内容 |
|---|---|
| **产品名称** | SpeakNow · AI 英语对话教练 |
| **产品定位** | 面向成人英语初学者的 AI 对话练习工具，通过模拟真实日常场景的开放式对话，帮助学习者在"用中学" |
| **目标用户** | 成人英语初学者（18 岁以上，CEFR A0-A1，词汇量 0-300 词） |
| **核心价值** | 低焦虑的真实对话体验 — AI 不会评判、不会不耐烦，随时可用 |
| **核心机制** | 场景化 AI 对话 + 脚手架式引导 + 隐式纠错反馈 + 多维度评分 |
| **产品形态** | 移动端 Web App（PWA），适配手机聊天界面 |

### 与现有产品的差异化

| 维度 | Duolingo | 流利说 | SpeakNow（本产品） |
|---|---|---|---|
| 核心交互 | 翻译/选择/拼写 | 跟读打分 | **开放式 AI 对话** |
| 输出类型 | 低（选词） | 中（朗读） | **高（自由表达）** |
| 反馈方式 | 对/错 | 发音分数 | **Recast + 自然推进** |
| 成人适配 | 游戏化偏儿童 | 内容偏商务 | **日常场景 + 低焦虑设计** |

## 二、学习者分析

### 成人初学者画像

| 维度 | 描述 |
|---|---|
| **年龄** | 18-45 岁，以 22-35 岁职场新人为主 |
| **教育背景** | 高中及以上，多数有大学学历 |
| **英语基础** | 学过但"说不出口"；词汇量 100-300；能认读简单单词但听说几乎为零 |
| **学习动机** | 出国旅游、简单工作交流、日常生活需要；工具型动机为主 |
| **学习特点** | 自律性强但面子敏感、怕犯错；偏好实用内容；碎片时间学习 |

### 成人 vs 儿童学习者的关键差异

| 维度 | 儿童学习者 | 成人初学者（本产品） |
|---|---|---|
| **认知能力** | 具体运算阶段 | 形式运算阶段，抽象思维成熟 |
| **语法学习** | 隐性习得为主 | 可借助显性规则理解 |
| **动机** | 兴趣驱动 | 实用需求驱动 |
| **焦虑源** | 失败/输 | **被评判、丢面子** |
| **内容偏好** | 游戏/故事 | **真实场景、即学即用** |
| **注意力** | 短（10-15min） | 中（15-25min） |

### 设计启示

1. **去游戏化**：不用积分/排行榜/卡通角色，用场景完成感和能力成长驱动
2. **零评判环境**：AI 不"扣分"，只提供自然的正确示范（Recast）
3. **即学即用**：每个场景学完可在真实生活中立即使用
4. **元认知支持**：场景结束后展示"你学到了什么"而不是"你错了多少"

## 三、学习目标（Bloom's Taxonomy）

### 产品总目标

By using SpeakNow, learners will be able to **handle 3 essential daily conversation scenarios in English** — ordering at a café, asking for directions, and checking into a hotel — with sufficient communicative competence to complete the task independently.

### 分层目标矩阵

| Bloom 层级 | 目标 | 实现方式 | 评估 |
|---|---|---|---|
| **记忆** | 认读并理解 45+ 日常场景核心词汇 | 场景词汇预览 + AI 对话中自然复现 | 对话中的词汇使用正确率 |
| **理解** | 听懂 AI 的提问和回应（可调节语速） | AI 语音输出 + 文字气泡双通道 | 用户回复的语义相关性 |
| **应用** | 在 AI 对话中使用目标句型完成交际任务 | 脚手架式引导 + 渐进式撤除支持 | 任务完成度评分 |
| **分析** | 识别 AI Recast 中的正确形式并自我修正 | AI 隐式纠错 + 用户自发修正 | 自我修正次数 |
| **综合** | 跨场景迁移：将场景 1 的句型灵活用于场景 3 | 共享句型在不同场景中复现 | 自由对话中的句型复用 |

## 四、AI 对话教学设计

### 4.1 对话脚手架策略

借鉴 Krashen (1982) 的「i+1」可理解输入理论与 Swain (1985) 的「可理解输出」假设：

| 阶段 | 脚手架程度 | AI 行为 | 示例 |
|---|---|---|---|
| **启航** | 高 — 提供完整句型模板 | AI 开场 + 展示可选回复（2-3 个预设选项） | AI: "Welcome! Would you like coffee or tea?" [选项: Coffee, please. / Tea, please. / I'd like a coffee.] |
| **练习** | 中 — 减少模板，增加开放度 | AI 追问开放问题 + 提供 1-2 个关键词提示 | AI: "What size would you like?" [提示: small / medium / large] |
| **应用** | 低 — 自由输入为主 | AI 提出开放问题，不提供选项 | AI: "Is there anything else you'd like?" [自由输入] |

### 4.2 难度自适应机制

```
         ┌─────────────┐
         │  用户回复    │
         └──────┬──────┘
                │
       ┌────────▼────────┐
       │  语义相关性检查  │
       └──┬───────────┬──┘
          │           │
    相关(>80%)    不相关(<80%)
          │           │
    ┌─────▼─────┐ ┌──▼──────────┐
    │ 语法评估   │ │ 温和重定向    │
    └──┬─────┬──┘ │ + 再次提供选项 │
       │     │    └──────────────┘
   准确 不准确
       │     │
  ┌────▼─┐ ┌▼──────────┐
  │自然推进│ │Recast + 推进│
  └──────┘ └────────────┘
```

### 4.3 纠错反馈策略

基于 Lyster & Ranta (1997) 的纠错反馈分类，本产品采用以下策略优先级：

| 优先级 | 策略 | 触发条件 | 对话流影响 |
|---|---|---|---|
| 1 | **Recast**（重述） | 语法小错但不影响理解 | 不中断对话，自然推进 |
| 2 | **Elicitation**（引导） | 使用了中文/错误词汇 | 暂停 1 轮，引导正确表达 |
| 3 | **Redirect**（重定向） | 回复完全偏离场景 | 暂停 1 轮，重新提供场景线索 |
| 4 | **Explicit**（显式纠错） | 同一错误出现 3+ 次 | 场景结束后的总结中展示 |

**关键原则**：对话流优先于纠错。只要交际意图可理解，AI 优先保持对话推进，纠错在自然对话中隐式完成。

### 4.4 对话示例（场景：At the Café）

```
🤖 AI: Welcome to Coffee Corner! What would you like to drink?
       [Coffee] [Tea] [Hot Chocolate]

👤 User: [点击] Coffee, please.

🤖 AI: Great choice! ☕ Would you like hot coffee or iced coffee?
       (recast: 自然重复 "coffee"，确认理解)

👤 User: [输入] hot

🤖 AI: Hot coffee — perfect for today! What size? 
       [Small] [Medium] [Large]
       (scaffold: 提供尺寸选项)

👤 User: [点击] Medium, please.

🤖 AI: Sure, a medium hot coffee. Anything else?
       (recast: 完整重述 "a medium hot coffee")

👤 User: [点击] No, that's all.

🤖 AI: That'll be $3.50. Thank you! ☕😊
       (任务完成 ✓)

─── 场景结束 ───
📊 任务完成: ⭐⭐⭐  |  语言准确: ⭐⭐  |  交际流畅: ⭐⭐⭐
💡 Recast 回顾: 「a medium hot coffee」— 注意形容词顺序
📝 新词汇: medium, iced, hot, size
```

## 五、场景库

### 场景 1 · ☕ At the Café

| 项目 | 内容 |
|---|---|
| **场景描述** | 在咖啡店点饮品和小食 |
| **核心词汇** | coffee, tea, hot chocolate, iced, hot, small, medium, large, sugar, milk, menu, how much |
| **目标句型** | I'd like a ___. / Can I have ___? / How much is it? / That's all. |
| **对话轮次** | 5-6 轮 |
| **任务目标** | 成功点一杯饮品，询问价格，完成支付 |

**对话流程图：**

```
问候 → 点饮品 → 确认温度/大小 → 是否需要其他 → 告知价格 → 结束
```

### 场景 2 · 🗺️ Asking Directions

| 项目 | 内容 |
|---|---|
| **场景描述** | 在陌生城市问路 |
| **核心词汇** | straight, turn left, turn right, next to, between, near, far, subway station, on the left, on the right, block |
| **目标句型** | Excuse me, where is ___? / Go straight. / Turn left/right. / It's on the left. / How far is it? |
| **对话轮次** | 5-6 轮 |
| **任务目标** | 成功获取从 A 点到 B 点的路线指引 |

**对话流程图：**

```
问路 → 确认目的地 → 路线描述（方向1） → 路线描述（方向2） → 距离确认 → 感谢结束
```

### 场景 3 · 🏨 Hotel Check-in

| 项目 | 内容 |
|---|---|
| **场景描述** | 在酒店前台办理入住 |
| **核心词汇** | reservation, passport, check in, room key, floor, elevator, breakfast, checkout, Wi-Fi, password |
| **目标句型** | I have a reservation. / Here is my passport. / What time is breakfast? / What's the Wi-Fi password? |
| **对话轮次** | 5-6 轮 |
| **任务目标** | 成功办理入住，获取房间号、早餐时间和 Wi-Fi 信息 |

**对话流程图：**

```
问候 → 确认预订 → 出示证件 → 告知房间信息 → 介绍设施 → 结束
```

## 六、评估设计

### 6.1 AI 自动评分维度

| 维度 | 权重 | 评分标准 | 数据来源 |
|---|---|---|---|
| **任务完成** | 40% | 3 分 = 完整完成任务 / 2 分 = 基本完成但有遗漏 / 1 分 = 需要 AI 多次重定向 | 对话流是否覆盖所有任务节点 |
| **语言准确** | 35% | 3 分 = 语法和词汇基本无误 / 2 分 = 有错误但不影响理解 / 1 分 = 错误多且影响理解 | AI 对每轮回复的语法分析 |
| **交际流畅** | 25% | 3 分 = 回复及时、用目标句型 / 2 分 = 偶有停顿或回避 / 1 分 = 大量使用单词而非句子 | 回复速度 + 句长 + 预设选项使用率 |

### 6.2 学习数据面板（面向用户）

场景结束后展示：
- ⭐ 总体评分（3 维度加权）
- 📝 本场景学到的词汇和句型清单
- 💡 AI 给出的 1-2 条改进建议
- 🔄 "再试一次" 按钮（重试成本低，鼓励迭代练习）

### 6.3 长期学习追踪（面向平台）

| 数据指标 | 用途 |
|---|---|
| 场景完成率 | 衡量内容难度是否合适 |
| 平均评分趋势 | 衡量学习进步 |
| 预设选项使用率 | 衡量从脚手架到自由输出的过渡 |
| 自我修正次数 | 衡量元认知和语言意识发展 |
| 高频错误列表 | 驱动个性化复习内容推荐 |

## 七、教学设计亮点

### 7.1 学习科学原则

| 原则 | 应用 |
|---|---|
| **任务型教学（TBLT）** | 每场景是一个真实交际任务，语言在完成任务中自然习得；任务完成度是首要评估维度 |
| **可理解输入（Krashen i+1）** | AI 根据用户水平动态调整语言复杂度；Recast 提供略高于用户当前水平的正确输入 |
| **可理解输出（Swain）** | 设计从选项到填空到自由输入的渐进路径，推动学习者产出完整句子 |
| **社会文化理论（Vygotsky ZPD）** | AI 充当"更有能力的他人"，在最近发展区内提供脚手架；渐进式撤除支持 |
| **成人学习理论（Knowles）** | 内容直接关联真实生活需求；学习者可自选场景、自定节奏；元认知反馈促进自主学习 |
| **低焦虑环境** | 无积分排名、无惩罚、无打断；AI 使用 Recast 而非显式纠错；重试成本为零 |

### 7.2 AI 教学策略亮点

| 策略 | 说明 |
|---|---|
| **Recast 优先** | 隐式纠错保护学习者的面子感和对话流畅度，符合成人学习心理需求 |
| **脚手架渐进撤除** | 同一场景可多次练习，每次 AI 提供的支持递减（选项→关键词→自由输入） |
| **跨场景句型复现** | "Can I have ___?" 在咖啡场景和酒店场景均出现，促进横向迁移 |
| **场景完成感** | 每个场景有明确的"任务完成"时刻（拿到咖啡/找到路/拿到房卡），给学习者即时的成就感 |

### 7.3 适宜 AI/教育科技平台的特长

- **NLU 驱动**：语义相关性判断不依赖关键词匹配，能理解不完美但交际有效的输入
- **自适应能力**：三条难度曲线（词汇复杂度、句式复杂度、脚手架程度）独立调整
- **数据闭环**：高频错误→个性化复习→再测试，形成完整的学习闭环
- **可扩展**：场景库可通过模板快速扩充（任何日常对话场景均可接入）
- **多模态**：文字 + 语音输出 + 语音输入（技术规划），覆盖听/说/读三技能

## 八、技术规格说明

### 8.1 AI 引擎功能需求

| 功能 | 说明 | 优先级 |
|---|---|---|
| **语义相关性判断** | 判断用户回复是否与当前对话上下文相关 | P0 |
| **语法准确度评估** | 对用户回复进行语法评分（不中断对话） | P0 |
| **Recast 生成** | 自动生成用户回复的正确/更自然版本 | P0 |
| **场景对话管理** | 维护对话状态，确保覆盖所有任务节点 | P0 |
| **难度自适应** | 根据用户前几轮表现调整后续提示的开方程度 | P1 |
| **语音识别（ASR）** | 支持语音输入转文字 | P1 |
| **语音合成（TTS）** | 生成自然的 AI 语音回复 | P1 |
| **发音评估** | 对语音输入进行发音准确度评分 | P2 |
| **个性化复习推荐** | 基于用户高频错误生成定制练习 | P2 |

### 8.2 技术架构建议

```
┌─────────────────────────────────────┐
│              前端 (Web App)          │
│  聊天 UI · 语音按钮 · 学习面板       │
└──────────────┬──────────────────────┘
               │ REST / WebSocket
┌──────────────▼──────────────────────┐
│           AI 对话引擎                │
│  ┌──────────┐  ┌──────────────────┐ │
│  │  NLU 模块 │  │  对话管理模块     │ │
│  │ 语义理解   │  │  场景状态机       │ │
│  │ 意图分类   │  │  任务节点追踪     │ │
│  └──────────┘  └──────────────────┘ │
│  ┌──────────┐  ┌──────────────────┐ │
│  │  评估模块  │  │  反馈生成模块     │ │
│  │ 语法评分   │  │  Recast 生成      │ │
```
